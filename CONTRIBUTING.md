# Contributing

Thank you for your interest in this project.

This project was created as a learning project to understand cloud monitoring on AWS. Suggestions, improvements, and bug reports are always welcome.

---

## How to Contribute

1. Fork this repository.
2. Create a new branch for your changes.

```bash
git checkout -b feature/your-feature-name
```

3. Make your changes.

4. Commit your changes.

```bash
git commit -m "Add your feature"
```

5. Push the branch.

```bash
git push origin feature/your-feature-name
```

6. Open a Pull Request.

---

## Contribution Guidelines

Please follow these simple guidelines:

- Keep the code clean and easy to understand.
- Use meaningful commit messages.
- Update the documentation if needed.
- Test your changes before creating a Pull Request.
- Follow the existing folder structure.

---

## Reporting Issues

If you find a bug or have a suggestion, please create a GitHub Issue with:

- A short description
- Steps to reproduce the issue
- Expected result
- Actual result
- Screenshots (if applicable)

---

## Suggestions

Ideas for improving the project are always welcome.

Examples include:

- Performance improvements
- Better monitoring scripts
- Additional AWS integrations
- Documentation improvements
- Dashboard enhancements

---

## Code of Conduct

Please be respectful and constructive when contributing. The goal of this project is to learn, improve, and share knowledge with others.

---

## Thank You

Thank you for taking the time to contribute to this project. Every suggestion, bug report, or improvement helps make the project better.
